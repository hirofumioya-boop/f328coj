### 各サービスの役割

| サービス | 役割 | URL | 費用 |
|---|---|---|---|
| **お名前ドットコム** | ドメイン（f328coj.jp）の管理 | onamae.com | 更新料のみ（2028/07/31まで） |
| **GitHub** | サイトのコード（HTMLファイル等）を保管 | github.com/hirofumioya-boop/f328coj | 無料 |
| **Vercel** | GitHubのコードを自動でサイトとして公開 | vercel.com | 無料 |
| **Web3Forms** | お問合せフォームのメール送信 | web3forms.com | 無料 |
| **Google Search Console** | Googleへのインデックス・SEO管理 | search.google.com/search-console | 無料 |

### 仕組みの流れ

```
① HTMLファイルを編集（PC上）
       ↓
② GitHubにアップロード（git push）
       ↓
③ Vercelが自動的に変更を検知
       ↓
④ 数分後に f328coj.jp に反映される
```

## 3. 日常的な更新作業

### 3-1. 更新情報を追加する

**ファイル**: `news-data.js`

このファイルだけ編集すれば、トップページ（最新5件）と更新情報ページ（全件）の両方に自動反映されます。

**手順:**

1. `news-data.js` をテキストエディタで開く
2. ファイルの先頭付近にある `NEWS_ITEMS = [` の直後に以下を追加：

```javascript
{
  date: "2026年○月",
  tag: "ツーリング",
  title: "タイトルをここに書く",
  body: "本文をここに書く。",
  link: { url: "/touring/report/2026-xxxx.html", text: "レポートを読む →" }
},
```

リンクが不要な場合は `link: null` にする。

3. GitHubにアップロード（後述の手順）

**タグの種類（目安）:**

| タグ | 使いどころ |
|---|---|
| `サイト` | サイト更新・お知らせ |
| `ツーリング` | ツーリングレポート追加 |
| `お知らせ` | その他のお知らせ |

---

### 3-2. イベントスケジュールを更新する

**ファイル**: `index.html`

`index.html` を開き、以下の部分を探して編集：

```html
<div class="schedule-list">
  <div class="schedule-item">
    <span class="schedule-date">1月18日</span>
    <span class="schedule-name">外房クラブツーリング</span>
  </div>
  <!-- ↑ この形式で追加・削除する -->
</div>
```

---

### 3-3. ツーリングレポートを追加する

1. `touring/report/` フォルダ内の既存ファイル（例: `2026-sotobou.html`）をコピー
2. 新しいファイル名で保存（例: `2026-karuizawa.html`）
3. 内容を書き換える
4. `touring/touring.html` の一覧カードに追加
5. `sitemap.xml` に新しいURLを追加
6. `news-data.js` に更新情報として追加（任意）
7. GitHubにアップロード

## 4. GitHubへのアップロード手順

ファイルを編集したら、以下の手順でGitHubにアップロードします。

### 方法A: GitHub Webサイトから直接アップロード（簡単）

1. https://github.com/hirofumioya-boop/f328coj にアクセス
2. 編集したいファイルをクリック
3. 右上の鉛筆アイコン（Edit）をクリック
4. 内容を編集
5. 下の「Commit changes」ボタンをクリック
6. 数分後に f328coj.jp に自動反映される

### 方法B: Claude Codeを使う（複数ファイルの編集に便利）

Claude Codeに「〇〇を修正して」と伝えると、自動でファイル編集＆GitHubへのアップロードまでやってくれます。

## 5. お問合せフォームのメールアドレス変更方法

1. https://web3forms.com にアクセス
2. 新しいメールアドレスを入力して「Create Access Key」
3. 届いたAccess Keyをコピー
4. `index.html` の以下の部分を新しいKeyに書き換え：

```html
<input type="hidden" name="access_key" value="ここを新しいKeyに変更" />
```

5. GitHubにアップロード

## 7. トラブルシューティング

### サイトが表示されない

1. https://f328coj.vercel.app にアクセスして表示されるか確認
2. 表示される → DNSの問題（お名前ドットコムの設定を確認）
3. 表示されない → Vercelの問題（vercel.comのダッシュボードを確認）

### お問合せメールが届かない

1. web3forms.com にログインして送信履歴を確認
2. 迷惑メールフォルダを確認
3. Access Keyが正しいか `index.html` で確認

### Googleで検索しても出てこない

- 新しいページを追加したら `sitemap.xml` に追記してGoogle Search Consoleから再送信
- 反映には数日〜1週間かかる場合があります
